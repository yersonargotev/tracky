# Domain docs

Tracky uses a single-context domain documentation layout:

- `CONTEXT.md` — project glossary and domain language.
- `docs/adr/` — architectural decisions.
- `docs/spikes/` — spike reports and evidence.

Agents should read `CONTEXT.md` and relevant ADRs before changing import, storage, categorization, or review workflows. Prefer the Spanish domain vocabulary already used in `CONTEXT.md` when naming product concepts.
